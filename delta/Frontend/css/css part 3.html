box model

this model just states that everything is in a box form ....every heading and every paragraph etc is in a box form .... it consists of:
1. content
2. padding
3.border
4. margin


-- in this the content is where we write or put the object like paragraph or image etc
-- then padding is the space between the blank content and border
--then border is the line till where the element occupies the space in the webpage
--then margin is the amount of space between the elements ...like if the images are simultaneous then the space between is determined by the margin attribute

height width

we can assign in pixels and percentage etc...its upon us.....if no content of inside ....like for say paragraph...then no height or width is alloted

border

there are different parameters
1.width
2. style
3. color

we can define these seperatly also like

h1{
    border-width: 2px;
    border-style: dashed;
    border-color: purple;
    width: 250px;
}

but this is lenghty and slow process so we assign these properties in one line like:

h1{
    border: width | style | color
}

border sides

to control an individual side of border:
border-left
border-right
border-top
border-bottom

example : border-left-style: dashed;

border radius

to make the round border give radius to the border in pixels or whatever to specify which corner to make round, specify the corner ....like: border-top-right-radius: 50px;

padding

like border padding can also be set side wise... ex - padding-top: 10px;
the padding shorthand can also be used
just declare size of the padding so it will set is evenly ...but if we declare it in a clockwise we can declare all the size at once ... for ex- padding: 2px 3px 4px 5px;
here we have declared all the sides

margin

is used to set the space outside the border....left right top bottom can also be set
use just 'margin' to set margin evenly
but as div is a block element so we cant decrease the right margin...so determine different sizes we can asiign them in clockwise order...
# br can also be set to 0 ( right margin remains though)

display property

block and inline property in this we can assign in inline element to behave like a block element and visa versa
ex -
h1{
    display: inline
}

span{
   display: block
} when a blosh element is converted into inline , the browser reads it as an inline element which leads the browser to treat them as inline , in which many block property cannot be held by inline elements....like height width and margin only works only horizontally

inline block

it is a display peroperty which allows the block element to behave like inline element and keep its block properties

display: none; can also be used to make something disappear..... it is usually used for testing purpose

relative unit

percentage

this is used to set size with respect to the parent element

em

this is used to set text related size.
'em' just means times or multiplied...like if the size is 20px then if i use 2em in the child element then the child will have twice its size .... if 1.5em then 1.5 times the size

rem

this is used to prevent the exponential grown of size in 'em' because it inherits from the one above, but in this 'rem' it only inherits form the parent
